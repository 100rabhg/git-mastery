## Learn Git Like You Actually Use It at Work

Git is not just a list of commands. It is a system for modeling change, collaboration, review, release, and recovery. This course helps you understand the commit graph, choose the right workflow, and recover safely when mistakes happen.


## What You Will Learn

- How Git stores history and why the commit graph matters
- How to use `status`, `add`, `commit`, `push`, `pull`, and `fetch` with confidence
- When to choose merge, rebase, revert, reset, or cherry-pick
- How to undo mistakes without damaging shared history
- How to design a Git branching strategy for your team
- How GitHub, GitLab, Bitbucket, Jira, and CI/CD fit into daily workflows

## Why This Course Stands Out

Most Git tutorials stop at command syntax. This course emphasizes decision-making:

- What changed in the graph?
- Is the branch shared?
- Are you rewriting history or preserving it?
- Are you solving a local mistake or a team coordination problem?

That framing makes advanced Git feel much more predictable.

## Suggested Learning Path

If you are new to Git, go in order from Foundations to Labs. If you already use Git but still get stuck during rebases, force pushes, or conflict recovery, focus on [History Manipulation](/git/history-manipulation), [Undo & Recovery](/git/undo-and-recovery), and the [Labs](/labs/).
