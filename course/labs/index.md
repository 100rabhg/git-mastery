# Git Labs and Exercises

These labs are designed to make Git judgment practical. Each one gives you a realistic scenario, a target outcome, and hints before the full approach becomes obvious.

## How to use the labs

- Create a throwaway practice repository
- Reproduce the scenario locally
- Pause before touching commands and sketch the desired graph
- Use `git status`, `git log --graph`, and `git reflog` often

## Lab Sequence

1. [Fix a broken commit](/labs/fix-broken-commit)
2. [Resolve a conflict](/labs/resolve-conflict)
3. [Move a commit to another branch](/labs/move-commit)
4. [Clean commit history](/labs/clean-history)

::: tip
If you want maximum learning, attempt the lab before reading any linked lesson again.
:::
